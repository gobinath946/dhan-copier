## Monitoring & Optimization

### **Key Metrics to Watch:**

1. **Master Score Distribution**
   - Should average 75-85 for entries
   - Higher = better quality setups

2. **AI Ensemble Agreement**
   - Entry: 4/5 or 5/5 is ideal
   - Exit: 2/3 or 3/3 is ideal
   - Higher agreement = higher confidence

3. **Algorithm Agreement Count**
   - Should be 7-10 for entries
   - Lower = mixed signals, avoid

4. **Win Rate**
   - Target: 65-75%
   - If lower, increase thresholds
   - If higher, can be more aggressive

5. **Average Hold Time**
   - Target: 15-20 seconds
   - Longer = not true scalping
   - Shorter = good execution

### **Optimization Tips:**

1. **Adjust Weights** in `masterAlgorithm.service.js`
   - Increase weight of best-performing algorithms
   - Decrease weight of noisy algorithms

2. **Adjust Thresholds**
   - Master Score: 75 → 80 (more conservative)
   - Confidence: 8 → 9 (higher confidence)
   - Agreement: 7 → 8 (more consensus)

3. **AI Model Selection**
   - `gpt-4o-mini`: Fast, cheap, good for scalping
   - `gpt-4o`: More accurate, slower, more expensive
   - `gpt-4-turbo`: Balance of speed and accuracy

4. **Backtest & Iterate**
   - Run for 1 week
   - Analyze performance
   - Adjust parameters
   - Repeat

## Troubleshooting

### **Issue: Too few trades**
- Lower Master Score threshold (75 → 70)
- Lower confidence threshold (8 → 7)
- Lower agreement requirement (7 → 6)

### **Issue: Too many losing trades**
- Increase Master Score threshold (75 → 80)
- Increase confidence threshold (8 → 9)
- Increase agreement requirement (7 → 8)

### **Issue: AI calls failing**
- Check OpenAI API key
- Check API rate limits
- Add retry logic
- Use fallback to rule-based

### **Issue: Slow execution**
- Reduce AI call frequency
- Use faster AI model (gpt-4o-mini)
- Cache algorithm results
- Optimize data fetching
